1) 위 코드의 수행 결과 d와 e는 항상 같은 결과를 출력한다. 그 이유는 무엇인가?

위 코드에서 e는 대입연산으로 d의 값을 복사하였다. 이는 d라는 문자가 가리키는 객체의 주소값을 가져와 그대로 e에 넣어준 것이다. 결국 d와 e가 동일한 객체를 가리키므로 그 출력값 또한 항상 같다.

2) 위 코드의 수행 결과 마지막 f의 내용은 다음과 같다. 이 때 'high' 키의 값 내용 중 500이 출력된 이유와 'low' 키의 값 내용 중 [1, 2, 3]이 출력된 이유를 설명하시오 ([hint]: shallow copy 관점에서 설명하시오.)

{'high': [4, 500, 6], 'low': [1, 2, 3]}

f는 d 사전을 copy() 메소드를 사용해 복사해온 객체이다. 때문에 f는 데이터를 추출해 새로운 문자열을 만드는 깊은 복사(deep copy)가 아닌 d 사전에서 키값과 가리키는 문자열의 주소만 복사하는 얕은 복사(shallow copy)의 결과값이다.

때문에 키값에 연결된 주소를 바꿔주지 않는 한, L1과 L2가 바뀌면 f의 결과값 또한 같이 바뀌게 된다.

        d['low'] = [10, 20, 30]
        d['high'][1] = 500

두 문장에서 d['low'] 에 대해 리스트를 대입한 연산은 사전d의 키값 'low'에 대응하는 주소를 바꿔주는 연산이다. 때문에 이미 주소를 복사해온 f에 대해서는 영향을 주지 않는다.

하지만 d['high'][1] = 500 문장은 d['high'] 의 결과값인 L2문자열에 대해 [1] 인덱싱 값을 바꿔주는 문장이므로 결과적으로는 문자열 L2의 원소를 바꾸는 것이다. 때문에 f를 출력하면 바뀐 원소값을 확인 할 수 있다.
